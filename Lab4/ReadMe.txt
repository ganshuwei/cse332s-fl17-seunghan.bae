===== Copy Control Design Decisions =========================================
To achieve the maximum functionality of the classes in this lab, I decided to provide all of the 5 copy control features to each of the classes.
In fact, during the course of completing this lab, I did not face an instance in which I needed any of those features.
This may be because there is only one game object created througout the execution of the program, and there is no need to copy/move it.
However, as I (or someone else) further develop the game and add features to it later, I might eventually need a well-defined copy control features.
Thus, copy/move constructors, copy/move-assignment operators, and destructors are provided for the TicTacToe, Gomoku, Sudoku, and UltimateTicTacToe classes.
For the GameBase class, I only provided copy/move constructors, since it is an abstract base class that other classes are derived from.
